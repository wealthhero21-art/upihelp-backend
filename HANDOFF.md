## Managing content (admin API)

All admin calls require header `x-admin-key: <ADMIN_API_KEY>`.

```bash
BASE=http://r12mfyt1bircueta7j4d36uy.217.216.58.194.sslip.io   # or the real domain once set

# Change the webview URL / onboarding / tabs (remote config):
curl -X PUT "$BASE/api/v1/admin/config" -H "x-admin-key: $KEY" -H 'content-type: application/json' \
  -d '{"platform":"all","data":{"webviewUrl":"https://www.upihelp.npci.org.in/","onboarding":[],"tabs":[],"featureFlags":{}}}'

# Set the update gate (force-update):
curl -X PUT "$BASE/api/v1/admin/app-versions" -H "x-admin-key: $KEY" -H 'content-type: application/json' \
  -d '{"platform":"android","minSupported":"1.0.0","latest":"1.1.0","storeUrl":"https://play.google.com/store/apps/details?id=com.valuegarageupihelp.app"}'

# Post an announcement:
curl -X POST "$BASE/api/v1/admin/announcements" -H "x-admin-key: $KEY" -H 'content-type: application/json' \
  -d '{"title":"Scheduled maintenance","body":"UPI Help may be slow 2-3am IST.","level":"warning"}'

# Inspect events:
curl "$BASE/api/v1/admin/events/stats" -H "x-admin-key: $KEY"
```

## Deploy / redeploy

Push to `main`, then trigger a deploy:
```bash
curl -X POST -H "Authorization: Bearer $COOLIFY_TOKEN" \
  "http://217.216.58.194:8000/api/v1/deploy?uuid=r12mfyt1bircueta7j4d36uy&force=false"
```
The container runs `prisma migrate deploy` then starts; defaults are seeded
idempotently on boot (`src/services/data.ts`).

## Quirks encountered (and fixes applied)

1. **Coolify injects `NODE_ENV=production` into the Docker build**, which made
   `npm ci` skip devDependencies → `tsc` failed (`Cannot find name 'process'`,
   missing `@types/node`). Fixed by pinning the builder stage to
   `NODE_ENV=development` + `npm ci --include=dev`, and removing the redundant
   `NODE_ENV` Coolify env var (the Dockerfile sets it for the runtime stage).
2. **`git push` used the wrong credential** when two `gh` accounts are logged in
   — `gh repo create --push` worked, but later plain pushes resolved to the
   inactive account (403). Push with the active account's token:
   `git push "https://x-access-token:$(gh auth token)@github.com/wealthhero21-art/upihelp-backend.git" main`.
3. **Coolify bakes all env vars into the image as build ARGs** for Dockerfile
   builds (Docker `SecretsUsedInArgOrEnv` warnings for `ADMIN_API_KEY` etc.).
   Acceptable on this private, single-tenant server, but ideally mark sensitive
   env vars as runtime-only in Coolify so they aren't written into image layers.
